# ADR-0051 — Unify pricing CEL to `llm_custom_total_cost`; retire per-model metadata keys

**Date:** 2026-06-15
**Status:** Accepted
**Deciders:** @stephane-segning
**Relates to:** [ADR-0028](./0028-owned-hardware-model-pricing.md) (pricing strategy),
[ADR-0021](./0021-burst-budget-billing-and-dual-plane-authconfigs.md) (rate-limit monthly budget),
[ADR-0005](./0005-per-user-attribution-via-authorino-headers.md) /
[ADR-0046](./0046-per-user-attribution-otlp-envelope-repair.md) (Loki access-log labels)

---

## Context

Each `ai-model` leaf chart ships an `AIGatewayRoute` that declares
`llmRequestCosts` — the per-field metadata the AIEG writes to Envoy's
dynamic metadata namespace `io.envoy.ai_gateway` after parsing the LLM
response.  In addition to the standard token-count keys (`llm_input_token`,
`llm_output_token`, `llm_total_token`, `llm_cached_input_token`), each route
declares a CEL expression that computes the per-request cost in micro-USD
(ADR-0028).

The Envoy core-gateway access log reads the cost back via the substitution
`%DYNAMIC_METADATA(io.envoy.ai_gateway:llm_custom_total_cost)%`, which is
the standard AIEG cost field (see `charts/core-gateway/templates/envoy-proxy.yaml`).
This field lands in every Loki access-log line and is what the per-user
Grafana dashboard queries as `gen_ai_usage_custom_total_cost` (after `| json`
dot-to-underscore sanitisation).

### The bug that introduced per-model keys

In an earlier version of the Envoy AI Gateway (AIEG), when multiple
`AIGatewayRoute` resources — one per model — all declared a `llmRequestCosts`
entry with the same `metadataKey: llm_custom_total_cost`, the AIEG or Envoy
XDS pipeline deduplicated or otherwise corrupted the key such that only one
route's value survived (or the values collided under concurrent requests).

A workaround was introduced in `charts/ai-model/templates/aigatewayroute.yaml`
**without an ADR** or inline comment explaining why: the pricing CEL's
`metadataKey` was changed from the standard `llm_custom_total_cost` to a
per-model key generated by the `ai-model.metadataKeySafe` Helm helper:

```
cost_<model_name>   # e.g. cost_glm_5p1, cost_qwen3_4b_local
```

The `BackendTrafficPolicy`'s monthly-budget rate-limit rule (`$costKey`) was
updated to match so rate limiting continued to work.

### Consequence of the workaround

The access log substitution continued to read from `llm_custom_total_cost`,
which was now never written.  Envoy substituted the empty metadata as `"-"`.
All cost fields in every Loki gateway stream were therefore the Envoy
placeholder `"-"`, making `unwrap gen_ai_usage_custom_total_cost | __error__=""`
silently drop every log line → **all cost panels in the per-user Grafana
dashboard showed "No data" since the workaround was introduced**, with no
visible error anywhere.  The silent failure was only discovered when the
dashboard was overhauled (2026-06-15) and each "No data" panel was debugged.

## Decision

The current version of Envoy used in this cluster resolves the multi-route
metadata deduplication issue.  All `AIGatewayRoute` resources now write the
pricing CEL result directly to the standard key `llm_custom_total_cost`:

```yaml
llmRequestCosts:
  - metadataKey: llm_input_token
    type: InputToken
  - metadataKey: llm_cached_input_token
    type: CachedInputToken
  - metadataKey: llm_output_token
    type: OutputToken
  - metadataKey: llm_total_token
    type: TotalToken
  - metadataKey: llm_custom_total_cost   # ← always this key; no per-model variant
    type: CEL
    cel: "..."
```

The `BackendTrafficPolicy` monthly-budget rule's `$costKey` is hardcoded to
`llm_custom_total_cost` (the template variable `$costKey` is kept for
readability but no longer calls `ai-model.metadataKeySafe`).

The `ai-model.metadataKeySafe` Helm helper (`charts/ai-model/templates/_helpers.tpl`)
is removed as it has no remaining callers.

## Consequences

**Positive**
- Cost data flows end-to-end: `AIGatewayRoute CEL → dynamic metadata →
  Envoy access log → Loki → Grafana`.
- Dashboard cost panels (`User — total cost`, `Overall — total cost`,
  `Top 15 users — cost`, model distribution by cost) now receive data.
- Single canonical key for both observability and rate limiting; no
  per-model branching logic in templates.

**Negative / Risk**
- If the AIEG multi-route deduplication issue reappears (e.g. after a future
  AIEG upgrade), cost values could be silently corrupted rather than
  definitively absent.  **Mitigation:** the per-user dashboard cost total
  should be periodically cross-checked against the token-count-derived
  estimate (`total_tokens × effective_rate`).  A significant discrepancy
  indicates a deduplication regression.

## What to do if the bug resurfaces

1. Introduce per-model keys again using a naming convention like
   `llm_cost_<metadataKeySafe(modelName)>` (write a NEW ADR superseding
   this one before merging).
2. Update the access log format in `charts/core-gateway/templates/envoy-proxy.yaml`
   to log ALL model-specific cost keys (one format.json field per model), or
   move to a sidecar aggregator.
3. Update Grafana queries to `sum` across the per-model cost fields in Loki.
