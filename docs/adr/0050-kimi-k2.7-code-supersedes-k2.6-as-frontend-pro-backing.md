# ADR-0050: Kimi K2.7 Code supersedes Kimi K2.6 as the `adorsys-frontend-pro` backing

**Status:** Accepted
**Date:** 2026-06-18
**Deciders:** @stephane-segning
**Relates to:** supersedes the K2.6 backing choice in [ADR-0044](0044-opencode-role-subagents-and-permission-scoped-tools.md) (branded `adorsys-*` aliases); price methodology unchanged from [ADR-0028](0028-price-self-hosted-models-at-cost-recovery.md)/[ADR-0021](0021-burst-budget-billing-and-dual-plane-authconfigs.md)

> **Amended by [ADR-0075](./0075-glm-5.2-price-drop-consolidate-glm-catalog.md) (2026-07-02):** the "do not adopt GLM-5.2" call below was priced against DeepInfra's $1.40/$0.25/$4.40 GLM-5.2 rate at the time. DeepInfra has since dropped GLM-5.2 to $0.93/$0.18/$3.00, undercutting GLM-5.1 on every axis — the catalog now standardizes on GLM-5.2 for every former GLM-5/GLM-5.1 backing. The Kimi K2.7 `adorsys-frontend-pro` decision below is unaffected. Body below is immutable.

## Context

ADR-0044 made each `adorsys-*` brand a stable alias whose backing model can be
swapped without users noticing (only the `info.displayName` parenthetical
reveals the backing). `adorsys-frontend-pro` was backed by **moonshotai/Kimi-K2.6**
— the heavier, multimodal frontend option (MoonViT vision encoder, "visual
inputs → UI").

Moonshot shipped **Kimi K2.7 Code** (DeepInfra `moonshotai/Kimi-K2.7-Code`,
launched 2026-06-12). Verified on DeepInfra:

| | Kimi K2.6 | Kimi K2.7 Code |
|---|---|---|
| Input / cached / output per 1M | $0.75 / $0.15 / $3.50 | **$0.74 / $0.15 / $3.50** |
| Context | 262,144 | 262,144 |
| Multimodal | yes | **yes** |
| Coding (Kimi Code Bench v2) | baseline | **+21.8%** |
| Program Bench | baseline | **+11.0%** |
| Reasoning tokens per task | baseline | **~30% fewer** |

K2.7 Code is a code-specialized successor that **strictly dominates K2.6** for
this role: a penny cheaper on input, identical cached/output price, the same
262k context, still multimodal (so it keeps serving the vision-capable frontend
role), and materially better at coding while burning ~30% fewer reasoning tokens
(lower real per-task cost). There is no axis on which K2.6 wins.

We also evaluated **GLM-5.2** (DeepInfra $1.40 / $0.25 / $4.40, 1M context, AA
Intelligence Index 51 vs GLM-5.1's 40) in the same pass. On the cache-heavy
blend its intelligence-per-dollar is **equal** to GLM-5.1 (~57) — it is a
*premium tier* (higher ceiling + 1M context at proportional cost), not a
better-value replacement. **Decision: do not adopt GLM-5.2 now** — the existing
GLM-5/GLM-5.1 backings remain the better value for routine planning/review, and
adding a premium brand was explicitly out of scope for this change.

## Decision

1. **Repoint `adorsys-frontend-pro`** from `moonshotai/Kimi-K2.6` to
   `moonshotai/Kimi-K2.7-Code` (both DeepInfra backends), update its pricing to
   $0.74 / $0.15 / $3.50 and `displayName` to *"Adorsys Frontend Pro (Kimi K2.7
   Code)"*. Budget cap unchanged (free $15 / pro $50, per ADR-0044).
2. **Retire the standalone `kimi-k2.6` catalog model**, replacing it with
   `kimi-k2.7-code` (same shape, new id/price/backing). Nothing else referenced
   `kimi-k2.6`.
3. **Do not add GLM-5.2** to the catalog (see Context).

This is a backing swap in the spirit of ADR-0044 — invisible to the well-known
and to the `frontend`/`@browser`/`@design` agent wiring (which pin
`adorsys-frontend`, not `-pro`); `adorsys-frontend-pro` is a user-selectable
brand, not an agent-pinned model.

## Consequences

- `adorsys-frontend-pro` users get a free quality upgrade (+coding, fewer tokens)
  at the same price and budget — no migration, the selected id is unchanged.
- The `kimi-k2.6` id disappears from the catalog. Any user who had explicitly
  selected `kimi-k2.6` (not the brand) must reselect `kimi-k2.7-code`. Acceptable
  — the brand alias is the supported, stable surface; raw model ids are not.
- GLM-5.2 remains available to revisit as a *premium brand* later (e.g.
  `adorsys-*-max`) if a 1M-context / max-reasoning need appears; it would be an
  addition, never a replacement of the GLM-5/5.1 value tiers.
- Verify with `helm template charts/ai-models` (route + BackendTrafficPolicy +
  budget bucket render for the renamed model). No well-known change needed.
